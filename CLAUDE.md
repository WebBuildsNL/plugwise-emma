## Project Overview

Homey app (Athom smart home platform) that adds Plugwise device support to Homey. Uses **Homey SDK v3** with the **Homey Compose** system. Targets Homey firmware >=12.4.0, local platform only. Category: climate.

## Commands

- **Lint:** `npm run lint` (ESLint with `eslint-config-athom/homey-app` rules)
- **Build/compose:** `homey app compose` (merges `.homeycompose/` fragments into the root `app.json` and driver manifests)
- **Run on Homey:** `homey app run` (deploys and runs the app on a connected Homey device)
- **Install on Homey:** `homey app install` (installs the app permanently)
- **Validate:** `homey app validate` (checks app structure and manifest correctness)

There are no test scripts configured.

## Architecture

### Homey Compose (`.homeycompose/`)

This is the **source of truth** for app configuration. Do NOT edit the root `app.json` directly — it is generated by `homey app compose` from `.homeycompose/app.json` plus fragments in subfolders.

Compose subdirectories (each contains JSON fragments that get merged during compose):
- `capabilities/` — custom device capabilities
- `discovery/` — device discovery strategies
- `drivers/settings/`, `drivers/templates/` — shared driver settings and templates
- `flow/triggers/`, `flow/conditions/`, `flow/actions/` — flow card definitions
- `signals/ir/`, `signals/433/`, `signals/868/` — signal definitions
- `screensavers/` — screensaver definitions
- `locales/` — app-level i18n strings

### Drivers (`drivers/`)

Each driver lives in `drivers/<driver-id>/` and typically contains:
- `device.js` — device logic (extends `Homey.Device`)
- `driver.js` — pairing logic (extends `Homey.Driver`)
- `driver.compose.json` — driver manifest fragment (merged by compose)

No drivers have been created yet.

### Entry Point

`app.js` — main app class extending `Homey.App`. Currently only logs initialization.

### Locales

`locales/en.json` — user-facing strings. Additional languages go in separate files (e.g., `nl.json`).

### Environment Variables

Secrets/credentials go in `env.json` (gitignored). Access at runtime via `Homey.env`.
